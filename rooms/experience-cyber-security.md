## What this room teaches

A wide-angle tour of the major cyber security disciplines so you can decide where to specialize. It covers offensive security, defensive security, governance/risk/compliance, application security, and a few specialty tracks. By the end you have a mental map of the field and a sense of which lane fits you.

## Key takeaways

- Cyber security isn't one job — it's at least a dozen distinct careers. The big buckets:
  - **Offensive** — penetration tester, red teamer, bug bounty hunter
  - **Defensive (Blue Team)** — SOC analyst, incident responder, threat hunter, detection engineer
  - **GRC** — governance, risk, compliance, audit
  - **AppSec** — secure coding, code review, DevSecOps
  - **Specialty** — cloud security, malware analysis, digital forensics, threat intelligence
- The difference between "I want to hack things" and "I want to defend things" is real and shapes the certs, tools, and day-to-day work in each lane.
- Salary, remote-friendliness, and growth varies sharply across the lanes. Defensive (especially SOC analyst → detection engineer → DFIR) tends to be the most remote-friendly entry path in 2026.

## What was new to me

- GRC is its own viable career — paperwork-heavy, but well-paid and accessible to people coming from non-technical backgrounds. Worth knowing about even if I'm not pursuing it.
- "Threat intelligence" sounds intimidating but at the entry level it's reading reports and tagging IOCs (indicators of compromise) so the SOC can use them. It's a natural Tier 2 step from SOC analyst, not a separate career path.

## How a SOC analyst would use this

- Even if I'm targeting SOC analyst, knowing what the adjacent roles do is interview gold. "Where do you see yourself in 3 years?" — having a credible answer like "Tier 2, then detection engineering" beats "I don't know yet" every time.
- When a SOC ticket needs help from another team (forensics, threat intel, GRC for breach disclosure), knowing what those teams actually do makes the handoff cleaner.

## Commands / concepts to remember

| Discipline | One-line summary |
|------------|------------------|
| SOC analyst | Front-line alert triage in a SIEM |
| Penetration tester | Authorized adversary simulation against scoped targets |
| Incident responder | Investigates and contains active incidents |
| Threat hunter | Proactively searches for threats not yet alerting |
| GRC | Governance, risk management, compliance |
| AppSec | Secure software development and code review |
| Threat intelligence | Tracks adversaries, TTPs, and IOCs |
